# Machine Configuration Files

This directory contains machine configuration files for different architectures (amd64, ppc64le, s390x) used to configure OpenShift nodes for Spyre workloads.

## Directory Structure

- `amd64/` - Machine configs for x86_64 architecture
  - `butane/` - Butane source files (human-readable format)
  - YAML machine config files
- `ppc64le/` - Machine configs for IBM Power (ppc64le) architecture
  - `butane/` - Butane source files (human-readable format)
  - YAML machine config files
- `s390x/` - Machine configs for IBM Z (s390x) architecture
  - `butane/` - Butane source files (human-readable format)
  - YAML machine config files

## Important: Updating ppc64le Machine Configs

**When you update any butane file in `ppc64le/butane/`, you MUST regenerate the machine config YAML files by running:**

```bash
make power-update-machine-configs
```

This command:
1. Converts the butane file (`99-worker-vfiopci.machineconfig.bu`) to YAML format (`99-worker-vfiopci.machineconfig.yaml`)
2. Updates the `99-vhostuser-bind.yaml` file with the base64-encoded content of `hack/bind-vfio.sh`

### Why This Is Required

The butane files are the source of truth for machine configurations. The YAML files are generated from these butane files and must be kept in sync. The `make power-update-machine-configs` target automates this conversion process using the butane tool.

## Synchronization with aiu-documentation Repository

**Important:** These machine configuration files and their documentation must be kept in sync with the `aiu-documentation` repository. When making changes here, ensure corresponding updates are made to the documentation repository to maintain consistency across the project.

## Machine Config Files

### amd64 Configurations

- `05-aiu-kernel-commandline.yaml` - Kernel parameters and VFIO-PCI driver settings
  - Source: `butane/05-aiu-kernel-commandline.bu`
  - Enables Intel and AMD IOMMU with passthrough mode
  - Loads vfio-pci and vfio_iommu_type1 kernel modules
  - Sets up VFIO device IDs for Spyre cards (1014:06a7, 1014:06a8)
  - Configures CRI-O memory limits
  - Sets up udev rules for VFIO devices

- `08-pciacs-v1.yaml` - PCIe Access Control Services (ACS) configuration
  - Source: `butane/08-pciacs-v1.bu`
  - Contains Perl script to disable PCIe ACS SrcValid for Spyre card branches
  - Runs as systemd service after kubelet starts
  - Required for proper PCIe device isolation and passthrough

- `09-vfstart.yaml` - SR-IOV Virtual Function creation
  - Source: `butane/09-vfstart.bu`
  - Applies to nodes with `vf` role label
  - Creates 2 virtual functions per Spyre physical function
  - Runs as systemd service to enable SR-IOV at boot
  - Sets permissions on /dev/vfio/* devices

- `mcp-vf.yaml` - Machine Config Pool for VF-enabled nodes
  - Defines a custom machine config pool for nodes that need SR-IOV VFs
  - Nodes must be labeled with `node-role.kubernetes.io/vf=""`

- `50-spyre-device-plugin-selinux-minimal.yaml` - SELinux configuration for the Spyre device plugin

### ppc64le Configurations

- `99-worker-vfiopci.machineconfig.yaml` - Configures VFIO-PCI driver settings for Spyre devices
  - Source: `butane/99-worker-vfiopci.machineconfig.bu`
  - Loads vfio-pci and vfio_iommu_spapr_tce kernel modules
  - Sets up VFIO device IDs for Spyre cards (1014:06a7, 1014:06a8)
  - Configures CRI-O memory limits
  - Sets up udev rules for VFIO devices
  - Configures memory lock limits for sentient group

- `99-vhostuser-bind.yaml` - Binds devices to VFIO driver
  - Contains base64-encoded bind-vfio.sh script
  - Auto-generated by `make power-update-machine-configs`

- `50-spyre-device-plugin-selinux-minimal.yaml` - SELinux configuration for the Spyre device plugin

### s390x Configurations

- `99-worker-vfiopci.machineconfig.yaml` - Configures VFIO-PCI driver settings for Spyre devices
  - Source: `butane/99-worker-vfiopci.machineconfig.bu`
  - Applies to nodes with `spyre` role label
  - Loads vfio-pci and vfio_iommu_type1 kernel modules
  - Sets up VFIO device IDs for Spyre cards (1014:06a7, 1014:06a8)
  - Configures CRI-O memory limits and DMA entry limits
  - Sets up udev rules for VFIO devices
  - Configures memory lock limits for sentient group

- `99-vhostuser-bind.yaml` - Binds devices to VFIO driver

- `mcp-spyre.yaml` - Machine Config Pool for Spyre-enabled nodes
  - Defines a custom machine config pool for s390x nodes
  - Nodes must be labeled with `node-role.kubernetes.io/spyre=""`

- `50-spyre-device-plugin-selinux-minimal.yaml` - SELinux configuration for the Spyre device plugin
